Demo Accounts

The example seeds one in-memory password user at startup.

  • Password login email: {{ demo_email }}
  • Password login password: {{ demo_password }}
  • Restarting the app resets the user store and all sessions.

The GitHub button uses a public mock GitHub OAuth server. Enter {{ demo_email }} there to link the seeded demo user, or any other email to create a new in-memory local user.

Emails starting with unverified are treated as unverified by the mock.

Flow

  • POST /login authenticates the demo user with CrossAuth.authenticate(...)
  • CrossAuth.login(...) writes the session cookie on the redirect response for password login
  • GET /auth/github/session/authorize starts the session-social login flow
  • /auth/github/callback handles the provider callback and resolves or creates the local user
  • /auth/github/session/callback consumes the internal auth code, creates the session cookie, and redirects back signed in
  • /profile and /api/me resolve the current user from that cookie
  • POST /logout clears the session again
