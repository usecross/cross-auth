How The Flow Is Chosen

  • /auth/github/session/authorize starts the session-social flow and ends with a browser cookie.
  • /auth/github/authorize starts the generic auth-code flow and ends with a local auth code for the SPA.
  • The same backend supports both without changing provider configuration.

Flow Summary

  • Session app: POST /login calls CrossAuth.authenticate(...) and CrossAuth.login(...).
  • Social session login: /auth/github/session/authorize redirects to GitHub, then /auth/github/session/callback resolves the local user and sets the session cookie.
  • Auth-server login: the SPA calls /auth/github/authorize, receives a local auth code at its own callback URL, then exchanges it at /auth/token.
